## 错误类型总结

- **表达可更聚焦**：仅要求「分析日志」但未说明希望得到的输出粒度（根因 / 堆栈解读 / 客户端改法 / 服务端修复方向）。

## 典型示例

### 分析 server.err 中 Whisper 语种报错

**原 query**：分析一下报错日志@`/Users/.../server.err`

**Refined query**：请根据 `server.err` 全文，定位首次 ERROR 的根因；说明触发请求的接口与参数；若与 OpenAI `language`（如 `zh-CN`）和底层 Whisper/mlx_whisper 语种代码不一致有关，请说明应对照文档使用何种代码，并区分「客户端改参数」与「服务端做规范化」两种修法。

**缺陷分析**：归类为表达可更聚焦——未指定希望从日志中得到根因、复现条件还是修复建议，助手可能只给摘要或漏掉可操作的参数对照。

**推荐理由**：明确「根因 + 接口 + 参数映射 + 修法选项」后，回答可直接用于改客户端或改服务端，减少来回追问。
